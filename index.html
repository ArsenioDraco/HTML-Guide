1 — Document basics

Minimal valid HTML document:

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Page title</title>
  </head>
  <body>
    <!-- content -->
  </body>
</html>

Always include lang, charset, and a responsive viewport. Use the HTML5 doctype.

2 — HTML document structure

Head vs Body

  • <head>: Metadata, styles, scripts (prefetch), title, meta description, link rels, canonical.
  • <body>: Visible content, scripts that manipulate DOM, main content, interactive elements.

Common head elements

<meta name="description" content="Short summary for search engines">
<meta name="author" content="Your Name">
<link rel="canonical" href="https://example.com/page">
<link rel="stylesheet" href="/styles.css">
<script defer src="/main.js"></script>
<meta property="og:title" content="Open Graph Title">
<meta property="og:image" content="/og-image.jpg">

3 — Text content & semantics

Use semantic elements instead of generic <div> and <span> when they convey meaning.

ElementUse
<header>Introductory content, site header
<nav>Navigation links
<main>Primary page content (one per page)
<article>Independent, self-contained composition (blog post)
<section>Thematic grouping of content
<aside>Sidebar content related to main content
<footer>Footer for a page or section
<figure> + <figcaption>Images with captions

Headings & outline

Use one <h1> per page (usually). Use heading levels to reflect document structure. Screen readers use them to navigate.

4 — Inline elements & text-level semantics

  • <a> — hyperlink (href, download, rel)
  • <em> — emphasis (typically italics)
  • <strong> — strong importance (typically bold)
  • <abbr> — abbreviation (title attribute)
  • <time> — machine-readable dates/times (datetime)
  • <mark> — highlighted text
  • <code>, <pre> — code samples and preformatted text
  • <small>, <del>, <ins>

Anchors: accessible links

<a href="/about" title="About this site">About</a>
      <a href="https://external.com" rel="noopener noreferrer" target="_blank">External</a>

<a href="#section-id" aria-label="Jump to section">Go to section</a>

5 — Forms & inputs

The modern HTML form includes semantic input types, validation attributes, and accessibility attributes.

Form elements cheat sheet

<form action="/submit" method="post" autocomplete="on" novalidate>
  <label for="name">Full name</label>
  <input id="name" name="name" type="text" required minlength="2" placeholder="Jane Doe">
<label for="email">Email</label>
  <input id="email" name="email" type="email" required>

  <label for="password">Password</label>
  <input id="password" name="password" type="password" minlength="8" autocomplete="new-password">

  <label for="age">Age</label>
  <input id="age" name="age" type="number" min="0" max="120">

  <label for="birthday">Birthday</label>
  <input id="birthday" name="birthday" type="date">

  <label>
    <input type="checkbox" name="subscribe"> Subscribe
  </label>

  <button type="submit">Submit</button>
</form>

Input types you should know

  • text, email, password, tel
  • url, number, range, date, time, datetime-local
  • search, color, file, checkbox, radio

Form accessibility tips

  • Always pair <label for> with inputs or wrap the input inside the label.
  • Use aria-describedby to link helpful hints or error messages.
  • Use semantic hints like autocomplete, inputmode, and proper type to aid mobile keyboards.
<label for="amount">Amount</label>
<input id="amount" name="amount" inputmode="decimal" pattern="^\d+(\.\d{1,2})?$" placeholder="e.g. 19.99">

<span id="amount-help">Round to two decimals</span>
<input aria-describedby="amount-help" aria-invalid="false">

6 — Multimedia

Images

<img src="/images/photo.jpg" alt="Descriptive alternative text" width="600" height="400" loading="lazy" decoding="async">

<figure>
  <img src="/images/chart.png" alt="Monthly sales chart">
  <figcaption>Sales by month</figcaption>
</figure>

Use alt text always. Use loading="lazy" for below-the-fold images.
