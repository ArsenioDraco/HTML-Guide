1 — Document basics

Minimal valid HTML document:

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Page title</title>
  </head>
  <body>
    <!-- content -->
  </body>
</html>

Always include lang, charset, and a responsive viewport. Use the HTML5 doctype.

2 — HTML document structure

Head vs Body

  • <head>: Metadata, styles, scripts (prefetch), title, meta description, link rels, canonical.
  • <body>: Visible content, scripts that manipulate DOM, main content, interactive elements.

Common head elements

<meta name="description" content="Short summary for search engines">
<meta name="author" content="Your Name">
<link rel="canonical" href="https://example.com/page">
<link rel="stylesheet" href="/styles.css">
<script defer src="/main.js"></script>
<meta property="og:title" content="Open Graph Title">
<meta property="og:image" content="/og-image.jpg">

3 — Text content & semantics

Use semantic elements instead of generic <div> and <span> when they convey meaning.

ElementUse
<header>Introductory content, site header
<nav>Navigation links
<main>Primary page content (one per page)
<article>Independent, self-contained composition (blog post)
<section>Thematic grouping of content
<aside>Sidebar content related to main content
<footer>Footer for a page or section
<figure> + <figcaption>Images with captions

Headings & outline

Use one <h1> per page (usually). Use heading levels to reflect document structure. Screen readers use them to navigate.

4 — Inline elements & text-level semantics

  • <a> — hyperlink (href, download, rel)
  • <em> — emphasis (typically italics)
  • <strong> — strong importance (typically bold)
  • <abbr> — abbreviation (title attribute)
  • <time> — machine-readable dates/times (datetime)
  • <mark> — highlighted text
  • <code>, <pre> — code samples and preformatted text
  • <small>, <del>, <ins>

Anchors: accessible links

<a href="/about" title="About this site">About</a>
      <a href="https://external.com" rel="noopener noreferrer" target="_blank">External</a>

<a href="#section-id" aria-label="Jump to section">Go to section</a>

5 — Forms & inputs

The modern HTML form includes semantic input types, validation attributes, and accessibility attributes.

Form elements cheat sheet

<form action="/submit" method="post" autocomplete="on" novalidate>
  <label for="name">Full name</label>
  <input id="name" name="name" type="text" required minlength="2" placeholder="Jane Doe">
<label for="email">Email</label>
  <input id="email" name="email" type="email" required>

  <label for="password">Password</label>
  <input id="password" name="password" type="password" minlength="8" autocomplete="new-password">

  <label for="age">Age</label>
  <input id="age" name="age" type="number" min="0" max="120">

  <label for="birthday">Birthday</label>
  <input id="birthday" name="birthday" type="date">

  <label>
    <input type="checkbox" name="subscribe"> Subscribe
  </label>

  <button type="submit">Submit</button>
</form>

Input types you should know

  • text, email, password, tel
  • url, number, range, date, time, datetime-local
  • search, color, file, checkbox, radio

Form accessibility tips

  • Always pair <label for> with inputs or wrap the input inside the label.
  • Use aria-describedby to link helpful hints or error messages.
  • Use semantic hints like autocomplete, inputmode, and proper type to aid mobile keyboards.
<label for="amount">Amount</label>
<input id="amount" name="amount" inputmode="decimal" pattern="^\d+(\.\d{1,2})?$" placeholder="e.g. 19.99">

<span id="amount-help">Round to two decimals</span>
<input aria-describedby="amount-help" aria-invalid="false">

6 — Multimedia

Images

<img src="/images/photo.jpg" alt="Descriptive alternative text" width="600" height="400" loading="lazy" decoding="async">

<figure>
  <img src="/images/chart.png" alt="Monthly sales chart">
  <figcaption>Sales by month</figcaption>
</figure>

Use alt text always. Use loading="lazy" for below-the-fold images.

Responsive & art-direction: <picture>

<picture>
  <source media="(min-width:1200px)" srcset="/images/hero-desktop.jpg">
  <source media="(min-width:600px)" srcset="/images/hero-tablet.jpg">
  <img src="/images/hero-mobile.jpg" alt="Hero image with people" loading="lazy">
</picture>

Video & audio

<video controls width="640" preload="metadata" poster="/thumb.jpg">
  <source src="/video.mp4" type="video/mp4">
  <source src="/video.webm" type="video/webm">
  Your browser does not support the video tag.
</video>
<audio controls>
  <source src="/audio.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

Always provide multiple sources for wide compatibility. Include captions/subtitles for accessibility (<track kind="captions">).

7 — Graphics: <canvas> & <svg>

SVG inline (recommended for icons & illustrations)

<svg viewBox="0 0 24 24" role="img" aria-label="Search icon" width="24" height="24">
  <path d="M15.5 14h-.79l-.28-.27A6.471 6.471 0 0016 9.5 6.5 6.5 0 109.5 16c1.61 0 3.09-.59 4.23-1.57l.27.28v.79l5 4.99L20.49 19l-4.99-5z"/>
</svg>

Canvas (bitmap drawing via JS)

<canvas id="c" width="600" height="400"></canvas>
<script>
const canvas = document.getElementById('c');
const ctx = canvas.getContext('2d');
ctx.fillStyle = '#2b6cb0';
ctx.fillRect(10,10,200,100);
</script>

Use SVG for vector icons and illustrations — scalable and accessible. Use canvas for pixel/painterly effects and games.

8 — Links, navigation & routing

10 — Accessibility & ARIA

Accessibility isn't optional. Use semantic HTML first, then ARIA only when necessary.

Basic rules

  • Use headings to create a logical document outline.
  • Provide alt for images; empty alt (alt="") for decorative images.
  • Use labels for form inputs.
  • Keep focus order meaningful and predictable.
  • Ensure contrast ratio is adequate (WCAG AA/AAA guidelines).

Common ARIA patterns

<button aria-pressed="false" aria-label="Toggle favorite">♡</button>

<nav role="navigation" aria-label="secondary">...

<div role="alert" aria-live="assertive">Error message</div>

11 — Progressive enhancement & best practices

  • Prefer semantic HTML; add JS progressively.
  • Defer non-critical scripts (defer or async where appropriate).
  • Serve responsive images, compress media, use preload/prefetch for critical assets.
  • Use feature detection (e.g., if ('serviceWorker' in navigator)).

12 — Microdata & structured data (schema.org)

Add structured data to help search engines understand your content. JSON-LD is recommended.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Amazing article title",
  "author": {"@type":"Person","name":"Your Name"},
  "datePublished": "2025-01-01",
  "image": ["https://example.com/og-image.jpg"]
}
</script>

13 — HTTP-related attributes & link relations

  • rel="noopener noreferrer" — security for external links.
  • rel="preload" as="script|style|image|font" — preload critical assets.
  • crossorigin — used for fonts & CORS requests.
  • integrity — Subresource Integrity for external scripts/styles.
<link rel="preload" href="/fonts/Inter.woff2" as="font" type="font/woff2" crossorigin>
<script src="https://cdn.example.com/lib.js" integrity="sha384-..." crossorigin="anonymous" defer></script>
